# Contributing

Thanks for your interest in contributing! We welcome bug reports, feature requests, and pull requests.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch: `git checkout -b my-feature`
4. Make your changes
5. Run tests (see below)
6. Commit with a clear message
7. Push and open a pull request

## Development Setup

See the README for installation and configuration instructions.

## Running Tests

```bash
# Python projects
python -m pytest tests/ -v

# Go projects
go test ./... -v
```

## Pull Request Guidelines

- Keep PRs focused — one feature or fix per PR
- Include tests for new functionality
- Update documentation if behavior changes
- Follow existing code style and conventions
- All CI checks must pass before merge

## Reporting Issues

- Use GitHub Issues for bugs and feature requests
- Include steps to reproduce for bug reports
- Include your OS, Python/Go version, and relevant config

## Code of Conduct

Be respectful and constructive. We're building something cool together.
